**游戏拆解案**

## 《布拉斯·库巴斯的死后委托》

**游戏灵感：《布拉斯·库巴斯死后的回忆》——马查多·德·阿西斯**

### 玩法设计

1.核心机制：**时间循环**

| 类别 | 子项 | 说明 |
| :--- | :--- | :--- |
| **核心机制** | 线索 | 线索性信息跨循环保留，利用先知优势追踪嫌疑人行踪、改变事态 |
| | 物品收集 | 物品每循环重置，需重新收集或规划替代路线 |
| | 角色日程 | 14位角色拥有独立时间表与路线，错过则本循环机会消失 |
| | 惩罚机制 | 非惩罚性失败：对话失误可等待下一循环重试，消除挫败感鼓励试错 |
| | 关系网 | 内置推理可视化工具，自动整理线索、物证、时间表与嫌疑人档案 |
| | 时间条与行动消耗 | 顶部时间条自然流逝，移动与对话消耗时间 |

2.主要玩法：玩家在循环的一天内询问14名嫌疑人获得线索、完成推理、指认真凶。


### 叙事设计

1.死者布拉斯·库巴斯作为委托人，委托玩家（即侦探）调查他的死因。

2.14名嫌疑人的人物关系网环环相扣，每名角色都有动机杀害死者。

3.碎片式叙事，将故事拆分至每名嫌疑人的询问以及与幽灵布拉斯·库巴斯的对话，从这些碎片信息中拼凑出嫌疑人与嫌疑人、死者与嫌疑人的关系。

4.开放式结局：游戏并未给出明确的凶手，而是将所有人都作为可能的行凶对象。笔者与制作者的想法一致，认为布拉斯·库巴斯，也就是死者本人即为真凶。

但就像制作者所说，答案并不重要。游戏传递的想法是：**死后一切皆为虚无**。因此，究竟是谁杀死了布拉斯·库巴斯，这是只有存在者才会在意的事，而作为游戏的叙事主角——死者布拉斯·库巴斯已经不在意这个答案了。
